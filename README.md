  # MLO考量多傳輸失敗原因的自適應傳輸策略
本模擬針對 IEEE 802.11be (Wi-Fi 7) 網路中，Access Point (AP) 管理多個Station (STA) 且存在多優先權流量需求的 BSS 環境進行效能提升，環境考慮密集部署下的多種傳輸失敗原因導致系統吞吐量降低，設計一套多鏈路的自適應傳輸策略。

## 概述
本模擬實驗評估 IEEE 802.11be 三頻段（2.4/5/6 GHz）下，採用Nonsimultaneous transmit and receive (NSTR)模式之 MLO 設備上行傳輸效能。核心機制藉由分析 AP 端的封包損壞特徵，並識別 STA 傳輸失敗的原因來動態調整傳輸策略，進而提升系統總吞吐量。

## 特色
- **上行傳輸失敗分析框架**:
  - 根據WiFi邊界傳輸之特性及OFDM技術下封包受損之物理特性，對三種較常見之傳輸失敗進行分析(Collision、 Hidden Node 、Noise)
  - 藉由AP分析通道傳輸失敗之原因，並採用週期性的Beacon封包來優化目前合適之傳輸策略
  - 提高高優先權封包之傳輸機會與吞吐量表現
