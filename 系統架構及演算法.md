# 系統架構
本研究所提出的傳輸失敗分類方法，區分Collision、HN與Noise所造成的傳輸失敗原因，並統計各類傳輸失敗的發生次數，當某種傳輸失敗的原因超過可容忍的閥值時，根據失敗原因採取相對應的處理策略，並週期性發送給STA進行策略更新。
![系統架構流程圖](https://github.com/wenjie55/IEEE-802.11be---Transmission-Strategys-for-MLO/blob/main/Images/%E5%9C%96%E5%9B%9B.png)

# 傳輸失敗分類
我們根據不同種類的傳輸失敗特徵進行錯誤分類，當AP接收後會判斷封包是否被成功接收，若接收失敗則會進一步判斷是否在符號偵測過程中同時觀察到First peak與Second peak產生;若未出現Second peak，則往下判斷Bit error 的錯誤率是否高於α%，用於區分Collision與Noise，並且我們會對封包傳輸後的分類結果進行統計。
![傳輸失敗原因分類流程圖](https://github.com/wenjie55/IEEE-802.11be---Transmission-Strategys-for-MLO/blob/main/Images/%E5%9C%96%E4%BA%94.png)

# 自適應傳輸策略抉擇
根據傳輸失敗的封包分類流程進行統計，並觀察一個固定視窗內的結果後，判斷目前通道發生的情況並決定是否啟用對應的處理策略，我們會先計算各種傳輸失敗佔總傳輸次數的比例，並據此判斷是否達到啟用特定策略的門檻。
![通道傳輸方式更新](https://github.com/wenjie55/IEEE-802.11be---Transmission-Strategys-for-MLO/blob/main/Images/%E5%9C%96%E5%85%AD.png)

# 更新傳輸策略
STA在接收到Beacon時用於更新這條通道上的傳輸策略，並根據AP在這段週期內觀察到各設備在通道上所傳輸之品質回報合適的通道更新資訊，並採用低複雜度方式來快速調整整體系統利用率。
